# Release v0.1.0

## Summary

`v0.1.0` documents the current SubspaceNetCC repository state as a research codebase for DOA estimation, SubspaceNet reproduction, channel cross-correlation input experiments, and CC-Toeplitz Root-MUSIC checks.

This release note does not report experimental performance results. It records implemented behavior and reproducibility constraints visible in the current code and test scripts.

## Main Changes

- `README.md` now describes the current project purpose, repository structure, setup options, default `main.py` configuration, minimal commands, implemented methods, limitations, and validation status.
- `CHANGELOG.md` records the `v0.1.0` documentation baseline.
- `docs/releases/v0.1.0.md` records release-specific reproducibility notes, validation attempts, known limitations, and next work.

## Implemented Behavior

- Snapshot-domain data generation is implemented through `SystemModelParams`, `SystemModel`, and `Samples`.
- Covariance-domain and autocorrelation-domain preprocessing are implemented through `create_cov_tensor`, `autocorrelation_matrix`, and `create_autocorrelation_tensor`.
- Channel cross-correlation-domain preprocessing is implemented through `channel_cross_correlation_matrix` and `create_channel_cross_correlation_tensor`.
- SubspaceNet-family models include `SubspaceNet`, `SubspaceNetEsprit`, `SubspaceNetCC`, `DeepRootMUSIC`, `DeepAugmentedMUSIC`, and `DeepCNN`.
- Traditional subspace methods include `MUSIC`, `RootMUSIC`, `Esprit`, and `MVDR`.
- CC-Toeplitz Root-MUSIC is implemented through `compute_zero_lag_cross_correlation`, `segmented_cross_correlation`, `toeplitz_reconstruction`, `CrossCorrToeplitzRootMUSIC`, and `CCToeplitzRootMUSIC`.
- Evaluation wiring includes SubspaceNet-family model evaluation, augmented subspace-method evaluation, model-based subspace-method evaluation, and the `cc-toeplitz-r-music` path.

## Reproducibility Notes

- `main.py` currently defaults to `SubspaceNetCC` with differentiable `root_music`.
- The default system parameters in `main.py` are `N=16`, `M=3`, `T=200`, `snr=10`, `signal_type="NarrowBand"`, and `signal_nature="non-coherent"`.
- The default CC parameters in `main.py` are `G=8`, `K=8`, `L=25`, and `ref_channel=0`.
- The default commands load data and evaluate a model, but do not create data, train a model, or save a model.
- The default data path is `data/datasets/uniform_bias_spacing`, and the default model-weight path is under `data/weights/final_models`.
- The Conda environment file includes `torch==2.0.1`; `pyEnv/requirements.txt` lists the same dependency as a commented line, so pip users need to install PyTorch separately.
- `data/`, datasets, outputs, model checkpoints, virtual environments, caches, and IDE metadata are ignored by `.gitignore` and should not be committed.

## Known Limitations

- The repository does not include committed datasets or model weights needed by the default `main.py` load-and-evaluate path.
- `main.py` uses source-code constants rather than CLI arguments or external config files.
- The default `samples_size` is `100000`, so full data generation and training can be expensive.
- `ModelGenerator.set_tau` returns immediately for `SubspaceNetCC`; the CC path uses `G`, `K`, `L`, and `ref_channel` instead of `tau`.
- `CCToeplitzRootMUSIC` requires `ref_channel=0` because the implementation assumes the first ULA channel provides the first Toeplitz covariance column.
- The current active Python environment is missing dependencies required for the provided CC check scripts.
